# `lazypcscfelicalite.h` 変更点まとめ

オリジナル版 (`lazypcscfelicalite.orig.h`) から、FeliCa Lite-S の高レベルなセキュリティ機能（MAC認証など）を Linux 環境および NDEF との共存環境で完全に活用するため、以下の機能拡張および修正を行いました。

## 1. MbedTLS を用いた Linux 環境向け暗号化対応
*   オリジナル版は Windows 環境（BCrypt）の暗号化実装しか持っていなかったため、Linux環境向けに `mbedtls/des.h` をインクルードし、MbedTLS を用いた DES / 3DES 暗号・復号化処理を新たに実装しました。
*   これにより、Linux上でもオリジナルの低レイヤ関数（`Key2des3`, `DualKey2des3` など）が機能するようになり、 FeliCa Lite-S の MAC_A（高度なMAC）を用いた相互認証機能などがフルサポートされました。

## 2. NDEF領域と秘密領域 (SPAD10〜13) の完全共存の実現
*   Android等のスマートフォンでNDEF（URL等）を読み込ませる際、従来は全ブロック（`Nmaxb=13`）をNDEF領域として申告していたため、一部のブロックがMACロックされていると全体が読み込みエラーになる仕様の衝突がありました。これを解決するため、NDEFヘッダ書き込み時のサイズ申告（`Nmaxb`）を `12 (0x0C)` に修正しました。
*   Androidからは「ブロック1〜12」までしか見えない構造となり、NDEFの挙動を全く阻害せずに、ユーザーデータ領域（`SPAD10〜13`）を完全に独立した「MAC認証が必須な秘密領域」として運用できるようになりました。
*   NDEFのフォーマットを、Androidが最も安定して認識する標準的な `Pure URI` フォーマットに変更しました。

## 3. SPAD10〜13 の「読み取り」MAC必須化による隠蔽と設定修正
*   オリジナル版の `enableSTATE` 関数では、ユーザー領域の書き込みロックフラグが不完全でした。これを `FF 3F FF 3F` を書き込むように修正し、ユーザー領域全体への書き込みにMACを必須化できるよう修正しました。
*   さらに、`enableSTATE` (`-W`) および `FirstIssue` (`-1`) 実行時に、`SPAD10〜13` の読み取り自体もMAC認証を必須とする設定（`MC_SP_REG_R_RESTR = 00 3C`）を自動で行うよう強化しました。
*   これにより、未認証のリーダー（一般のスマホ等）からは `SPAD10〜13` にアクセスできなくなり、完全にセキュアな秘密データ領域として運用できるようになりました。

## 4. MCブロックのバグ修正およびデバッグ表示の適正化
*   `FirstIssue` において、`SYS_OP` (NDEF有効化等) と `RF_PRM` (通信パラメータ) の書き込み位置がずれていたオリジナルのバグを修正しました（正しくは `mc[3]` が `SYS_OP` で `mc[4]` が `RF_PRM`）。
*   FeliCa Lite-S の隠しブロック（`0x0080`〜`0x00A0`等）の名前定義とデバッグ用ダンプ機能を追加し、ステータス確認コマンド（`-s`）でカードの内部状態（ロック状況や鍵バージョンの有無など）を詳細に表示できるように改善しました。
*   カードへ書き込む際に読み取られる書き込み回数カウンタ（WCNT）の内部値が、標準出力に漏れ出てしまう問題（`CC FF FF 00 00 00 00 00` など）を修正し、デバッグモード（`-d`）指定時のみ出力されるように `if(debugFlag)` で囲みました。

## 5. NDEF読み取り処理の堅牢化
*   `readNdefContent` において、不正なデータを持つタグを読み込んだ際のバッファオーバーランを防ぐ安全対策 (`payload_len > 250` の境界チェック) を追加しました。
