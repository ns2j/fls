# fls_ws の Windows サービス化ガイド

このガイドでは、オープンソースのサービスラッパーツール **WinSW (Windows Service Wrapper)** を使用して、`fls_ws.exe` (WebSocketサーバー) をWindowsのバックグラウンドサービスとして登録し、PC起動時に自動で立ち上がるようにする手順を説明します。

## 導入手順

### 1. WinSW のダウンロード
GitHubの [WinSW Releases ページ](https://github.com/winsw/winsw/releases) にアクセスし、最新バージョンの実行ファイル（例: `WinSW-x64.exe`）をダウンロードしてください。

### 2. ファイルの配置とリネーム
ダウンロードした `WinSW-x64.exe` を、本プロジェクトの `fls_ws.exe` と同じフォルダーに配置します。
配置後、ファイル名をわかりやすく **`fls_ws_service.exe`** に変更してください。

### 3. 設定ファイルの準備
本リポジトリに同梱されている `fls_ws_service.sample.xml` をコピーし、ファイル名を **`fls_ws_service.xml`** に変更してください。

コピーした `fls_ws_service.xml` をテキストエディタで開き、以下の `<executable>` タグ内のパスを、**ご自身の環境の `fls_ws.exe` の絶対パス（フルパス）** に書き換えて保存してください。

### 4. サービスのインストールと起動
管理者権限でコマンドプロンプトまたはPowerShellを開き、ファイルのあるフォルダーに移動（`cd`）して以下のコマンドを実行します。

```cmd
# サービスのインストール（Windowsにサービスとして登録されます）
.\fls_ws_service.exe install

# サービスの起動
.\fls_ws_service.exe start
```

これだけでサービス化は完了です！
以降はPCを起動するたびにバックグラウンドで `fls_ws` が実行されます。

---

## アンインストール方法
サービスを停止し、登録を解除したい場合は、同様に管理者権限のコマンドプロンプトから以下を実行してください。

```cmd
.\fls_ws_service.exe stop
.\fls_ws_service.exe uninstall
```

## トラブルシューティング（カードリーダーが認識されない場合）
Windowsサービスはデフォルトでシステム固有のアカウント (`LocalSystem`) として動作します。
もしサービスとして起動した際にFeliCaリーダー（PaSoRi等）が認識されない、または通信エラーが発生する場合は、サービスを実行するユーザー権限を普段お使いのアカウントに変更することで解決します。

1. `Win + R` キーを押し、`services.msc` と入力して「サービス」管理画面を開きます。
2. リストから `FeliCa WebSocket Server` を探してダブルクリックします。
3. **「ログオン」** タブを開きます。
4. 「ローカル システム アカウント」から **「アカウント」** にチェックを変更します。
5. 普段ログインしているWindowsのユーザー名（とパスワード）を入力し、「適用」を押します。
6. サービスを再起動します。
